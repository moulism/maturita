Úvod do SQL a základní příkazy

⏱ ~15 minut
📝 Potítko – napiš si na papír
  • SQL = standardizovaný jazyk pro relační DB
  • DDL → CREATE, ALTER, DROP (stavba struktury)
  • DML → SELECT, INSERT, UPDATE, DELETE (data)
  • DCL → GRANT, REVOKE (práva)
  • SELECT pořadí: SELECT – FROM – WHERE – ORDER BY
  • DISTINCT = unikátní hodnoty, TOP/LIMIT = omezení, OFFSET = stránkování

Co je SQL a databázový systém

SQL (Structured Query Language) je standardizovaný jazyk pro komunikaci s relačními databázemi. Databázový systém neslouží jen k uložení dat jako Excel — funguje jako centrální mozek firmy. Zajišťuje, že tisíce zaměstnanců mohou bezpečně a zároveň pracovat s evidencí zákazníků, objednávek nebo skladových zásob, aniž by se data poškodila nebo přepisovala.

Rozdělení příkazů SQL — 3 podjazyky

NázevZkratkaCo děláPříkazyBusiness využití
Data Definition LanguageDDLStavba struktury („regály")CREATE, ALTER, DROPProgramátor navrhuje tabulky pro e-shop
Data Manipulation LanguageDMLPráce s daty uvnitřSELECT, INSERT, UPDATE, DELETEÚčetní hledá objednávky, mění stav faktury
Data Control LanguageDCLPráva a bezpečnostGRANT, REVOKEAdmin: skladník vidí jen sklad, ne platy
🏗️ Analogie — stavba obchodního domu DDL = architekt: staví zdi a regály (CREATE TABLE). DML = zaměstnanci: plní regály zbožím (INSERT, SELECT). DCL = bezpečnostní služba: určuje, kdo má od čeho klíče (GRANT). Pokud použiji DELETE, smažu krabici z regálu. Pokud DROP, zbourám celý regál i s obchodním domem.

Struktura příkazu SELECT

SELECT je nejpoužívanější příkaz — slouží ke čtení dat. Musí dodržovat přesné pořadí klíčových slov:

SELECT   jmeno, prijmeni          -- Co chci zobrazit (projekce)
FROM     Zakaznici               -- Z jaké tabulky čerpám
WHERE    mesto = 'Praha'         -- Filtrování řádků
ORDER BY obrat DESC             -- Řazení (DESC = sestupně)

Klíčová slova pro úpravu výstupu

DISTINCT
Odstraní duplicity. SELECT DISTINCT produkt — každý produkt jen jednou, i když je v tabulce tisíckrát.
TOP / LIMIT
Omezí počet vrácených řádků. SELECT TOP 10 zobrazí jen 10 nejlepších zákazníků.
OFFSET + FETCH
Stránkování: OFFSET 50 ROWS FETCH NEXT 20 ROWS ONLY — přeskoč prvních 50, ukaž dalších 20. Používá se na e-shopech, kde nechceme zobrazit milion produktů najednou.
ORDER BY
ASC = vzestupně (výchozí), DESC = sestupně. Lze řadit víceúrovňově (nejdřív podle města, pak podle příjmení).
⚠️ Chytáky a záludné otázky
  • „Jaký je rozdíl DDL a DML?" — DDL mění strukturu (tabulky, sloupce). DML mění data uvnitř. DROP zbourá tabulku. DELETE smaže řádky uvnitř tabulky.
  • „Proč mít databázi a ne jen Excel?" — Databáze: souběžný přístup tisíců uživatelů, referenční integrita, transakce (rollback), bezpečnostní práva, miliardy řádků. Excel: max. 1M řádků, žádná souběžnost, žádná ochrana dat.
  • „Co je DCL?" — Studenti zapomínají! Data Control Language — GRANT (přiděl práva), REVOKE (odeber práva). Určuje, kdo co smí vidět a měnit.
  • „V jakém pořadí databáze zpracovává dotaz?" — Nezačíná od SELECT! Pořadí: FROM → WHERE → GROUP BY → HAVING → SELECT → ORDER BY. Proto nelze ve WHERE použít alias definovaný v SELECT.
  • „Co je stránkování a proč se používá?" — Na e-shopu nechceme zobrazit milion produktů najednou. OFFSET přeskočí prvních N produktů, FETCH zobrazí dalších M. Šetří paměť serveru i prohlížeče.
💡 Tip na 15 minut — říkej komisi „Rád bych detailněji popsal rozdíl mezi DDL a DML na konkrétní analogii. Představte si stavbu obchodního domu. DDL příkazy (CREATE, ALTER) používá architekt — staví zdi a regály. V tento moment je obchod prázdný. Jakmile je hotovo, nastupují zaměstnanci s DML příkazy (INSERT, UPDATE, SELECT) — plní regály zbožím. Pokud použiji DML příkaz DELETE, smažu krabici z regálu. Pokud ale použiji DDL příkaz DROP, zbourám celý regál i s obchodním domem — a žádná data se nedají vrátit."
