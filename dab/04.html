Práce s výrazy – CASE, CAST, CONVERT, pivotace

⏱ ~15 minut
📝 Potítko – napiš si na papír
  • CAST = konverze datového typu (text → číslo)
  • CASE = IF-THEN-ELSE v SQL
  • Jednoduchý CASE = přesná shoda | Hledaný CASE = podmínky (<, >)
  • Pivotace = přetočení řádků na sloupce pomocí CASE + SUM
  • Hromadný INSERT = VALUES (1,'a'),(2,'b'),(3,'c') — místo tisíce dotazů

Převody datových typů — CAST a CONVERT

V databázi nelze matematicky sečíst slovo a číslo. Při importu dat ze starých systémů bývá cena uložena jako text (VARCHAR). CAST nebo CONVERT řeknou databázi: „Nyní se na tento text dívej jako na číslo."

-- CAST: standardní SQL
SELECT CAST(cena_text AS DECIMAL(10,2)) * 1.21 AS cena_s_dph
FROM   Produkty

-- CONVERT: specifické pro MS SQL (lze zadat formát data)
SELECT CONVERT(VARCHAR, datum, 104) AS datum_cz   -- formát DD.MM.YYYY

Podmíněná logika — CASE

CASE funguje jako IF-THEN-ELSE přímo v SQL dotazu. Vytvoří virtuální sloupec s hodnotou podle podmínky.

Jednoduchý CASE — porovnává přesné hodnoty:

SELECT jmeno,
       CASE znamka
           WHEN 1 THEN 'Výborný'
           WHEN 2 THEN 'Chvalitebný'
           ELSE 'Dostatečný a níže'
       END AS slovni_hodnoceni
FROM   Studenti

Hledaný CASE — umožňuje podmínky s operátory:

SELECT jmeno, utrata,
       CASE
           WHEN utrata > 100000 THEN 'VIP'
           WHEN utrata > 50000  THEN 'Gold'
           ELSE 'Standard'
       END AS kategorie
FROM   Zakaznici

Pivotace pomocí CASE — řádky na sloupce

Pokud máme prodeje zapsané v řádcích (Leden, Únor, Březen...) a chceme přehled pro vedení, kde každý měsíc je samostatný sloupec:

SELECT
    SUM(CASE WHEN mesic = 'Leden'  THEN cena ELSE 0 END) AS Leden,
    SUM(CASE WHEN mesic = 'Unor'   THEN cena ELSE 0 END) AS Unor,
    SUM(CASE WHEN mesic = 'Brezen' THEN cena ELSE 0 END) AS Brezen
FROM Prodeje

Hromadný INSERT — řádkové konstruktory

Místo tisíce jednotlivých INSERT příkazů vložíme vše najednou — šetří to výkon databáze:

INSERT INTO Produkty (id, nazev, cena) VALUES
    (1, 'Notebook', 25000),
    (2, 'Myš',      500),
    (3, 'Klávesnice', 1200)
⚠️ Chytáky a záludné otázky
  • „Kdy CAST selže?" — Když text nelze převést na číslo (obsahuje písmena). CAST('abc' AS INT) hodí chybu. V MS SQL lze použít TRY_CAST, který vrátí NULL místo chyby.
  • „Jaký je rozdíl jednoduchý vs. hledaný CASE?" — Jednoduchý: přesná shoda (WHEN hodnota). Hledaný: podmínky s operátory (WHEN x > 100). Hledaný je flexibilnější.
  • „Co je pivotace a proč ji neřešit v Excelu?" — Pivotace v SQL pracuje přímo na serveru nad miliony řádků. Exportovat milion řádků do Excelu jen pro pivot je pomalé a zbytečně zatěžuje síť.
💡 Tip na 15 minut — říkej komisi „CASE je naprosto nedocenitelný nástroj pro reporting. Marketingové oddělení chce zákazníky rozdělené podle věku. Bez CASE bychom museli data exportovat do Excelu. Pomocí CASE vytvoříme v SQL dotazu virtuální sloupec: CASE WHEN vek < 18 THEN 'Mládež' WHEN vek BETWEEN 18 AND 65 THEN 'Dospělí' ELSE 'Senioři' END. Databáze to roztřídí za zlomek vteřiny přímo na serveru."
