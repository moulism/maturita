T-SQL a uložené procedury

⏱ ~15 minut
📝 Potítko – napiš si na papír
  • T-SQL = SQL + proměnné, IF/ELSE, WHILE, TRY-CATCH (MS SQL Server)
  • Stored Procedure = pojmenovaný blok SQL kódu uložený v DB
  • CREATE PROCEDURE jmeno AS BEGIN ... END | EXEC jmeno
  • Transakce: BEGIN TRAN → test → ROLLBACK nebo COMMIT
  • Trigger = procedura spouštěná automaticky po INSERT/UPDATE/DELETE

Co je T-SQL — rozšíření standardního SQL

Standardní SQL umí jen základní operace s daty. T-SQL (Transact-SQL) je rozšíření od Microsoftu pro MS SQL Server, které z SQL dělá plnohodnotný programovací jazyk. Přidává:

  • Proměnné: DECLARE @pocet INT = 0
  • Podmínky: IF @pocet > 0 BEGIN ... END ELSE BEGIN ... END
  • Cykly: WHILE @i <= 10 BEGIN SET @i += 1 END
  • Zachytávání chyb: BEGIN TRY ... END TRY BEGIN CATCH ... END CATCH

Uložená procedura (Stored Procedure)

Je to pojmenovaný blok SQL kódu uložený přímo v databázi. Účetní nemusí umět SQL — jen klikne na tlačítko, aplikace spustí proceduru a ta vygeneruje report.

-- Vytvoření procedury
CREATE PROCEDURE Mesicni_Souhrn @mesic INT, @rok INT
AS
BEGIN
    SELECT zakaznik, SUM(cena) AS trzby
    FROM   Faktury
    WHERE  MONTH(datum) = @mesic AND YEAR(datum) = @rok
    GROUP BY zakaznik
END

-- Spuštění procedury
EXEC Mesicni_Souhrn @mesic = 5, @rok = 2025

Testování procedur pomocí transakcí

Hromadné operace se musí testovat nanečisto. Transakce umožňuje buď vše potvrdit (COMMIT) nebo vrátit zpět (ROLLBACK):

BEGIN TRAN                          -- spusť nanečisto

UPDATE Produkty SET cena = cena * 1.10  -- zdraž vše o 10 %

SELECT * FROM Produkty                -- zkontroluj výsledek

-- ROLLBACK    -- pokud je špatně, vrať zpět
COMMIT                              -- pokud OK, potvrď

Triggery — automatické spuštění

Trigger je speciální procedura, která se spustí automaticky po INSERT, UPDATE nebo DELETE. Zákazník klikne „Koupit" → do DB se vloží objednávka (INSERT) → trigger automaticky odečte kus ze skladu a odešle e-mail skladníkovi.

CREATE TRIGGER po_objednavce
ON Objednavky
AFTER INSERT
AS
BEGIN
    UPDATE Sklad SET mnozstvi -= 1
    WHERE  produkt_id = (SELECT produkt_id FROM inserted)
END
⚠️ Chytáky a záludné otázky
  • „Proč procedury místo přímého SQL?" — Bezpečnost! Manažer nemá přímý přístup k tabulce platů. Smí spustit jen proceduru Pridej_Premie. Nelze číst ani měnit data libovolně.
  • „Jaký je rozdíl procedura a trigger?" — Proceduru spustí uživatel ručně (EXEC). Trigger se spustí automaticky jako reakce na událost v tabulce.
  • „Co ROLLBACK vs COMMIT?" — ROLLBACK = vrátit vše zpět jako by se nic nestalo. COMMIT = natrvalo potvrdit změny. Bez COMMIT nebo ROLLBACK transakce čeká a blokuje přístup ostatním!
  • „Proč TRY-CATCH v proceduře?" — Zachytí chybu za běhu a místo pádu celé procedury provede záložní akci (zápis do logu chyb, ROLLBACK).
💡 Tip na 15 minut — říkej komisi „Procedury mají v byznysu obrovskou výhodu z pohledu bezpečnosti. Místo přímého přístupu k tabulce platů zakážu manažerům přímá práva a vytvořím proceduru 'Pridej_Premie'. Manažer může spustit jen tuto proceduru — a tím garantuji, že s daty stane jen přesně to, co jsem jako programátor povolil uvnitř procedury."
