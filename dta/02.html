Návrh třídy Transform s logováním

⏱ ~15 minut
📝 Potítko – napiš si na papír
  • Třída = šablona/blueprint; __init__ = konstruktor, volá se automaticky při vytvoření objektu
  • self.logger = logging.getLogger(__name__) – inicializuj VŽDY v __init__
  • Úrovně: DEBUG < INFO < WARNING < ERROR < CRITICAL
  • try/except: v except bloku vždy self.logger.error(...) + raise
  • Metody třídy přijímají self jako první parametr

Proč třídy a OOP?

Třída (class) sdružuje data (atributy) a funkce (metody) do jednoho celku. Pro transformaci dat: místo mnoha volných funkcí máme jeden objekt Transform, který si pamatuje svůj stav (logger, cestu k souboru, dataframe) a nabízí jasné rozhraní.

Základní struktura třídy Transform

import logging
import pandas as pd

class Transform:
    def __init__(self, source: str):
        self.logger = logging.getLogger(__name__)
        self.source = source
        self.df = None

    def load(self) -> None:
        try:
            self.df = pd.read_csv(self.source, sep=';', encoding='utf-8-sig')
            self.logger.info(f"Načteno {len(self.df)} řádků")
        except FileNotFoundError as e:
            self.logger.error(f"Soubor nenalezen: {e}")
            raise

    def add_vat(self, col: str, rate: float = 0.21) -> None:
        try:
            self.df[col + '_s_dph'] = self.df[col] * (1 + rate)
            self.logger.info(f"DPH přidáno ke sloupci {col}")
        except KeyError as e:
            self.logger.error(f"Sloupec neexistuje: {e}")
            raise

t = Transform('data.csv')
t.load()
t.add_vat('cena')

Konfigurace logování

logging.basicConfig(
    level=logging.DEBUG,
    format='%(asctime)s %(name)s %(levelname)s %(message)s',
    handlers=[
        logging.FileHandler('transform.log'),
        logging.StreamHandler()
    ]
)

Úrovně logování

ÚroveňKdy použít
DEBUGLadění – podrobné informace pro vývojáře
INFONormální průběh: data načtena, transformace proběhla
WARNINGNeočekávaná situace, ale program běží dál
ERROROperace selhala (loguj + raise)
CRITICALKritická chyba – program se nemůže zotavit
⚠ Chytáky a záludné otázky
  • Proč nestačí print() místo loggingu? – print() nejde selektivně vypnout, neredirektuje do souboru, nemá časová razítka ani úrovně závažnosti. Logger se konfiguruje bez změny kódu (DEBUG v testování, WARNING v produkci).
  • Co je rozdíl konstruktor vs metoda? – __init__ se volá automaticky při t = Transform(). Ostatní metody (load, add_vat) voláš explicitně.
  • Proč raise v except bloku? – Chceme zalogovat chybu A zároveň ji propagovat výš. Bez raise by program pokračoval s prázdnými daty – tiché selhání je horší než hlasitá chyba.
💡 Tip na 15 minut — říkej komisi Návrh třídy s logováním ukazuje princip zapouzdření. Klíčový detail, který učitelé sledují: logger inicializuji v __init__ a v každém except bloku volám self.logger.error() ještě před tím, než výjimku hodím dál přes raise. Tak mám v logu vždy přesný záznam o tom, co a kde selhalo.
