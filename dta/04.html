Otestování transformační funkce pomocí pytest

⏱ ~15 minut
📝 Potítko – napiš si na papír
  • Test = funkce začínající test_ v souboru začínajícím test_
  • assert výraz – projde pokud True, AssertionError pokud False
  • pytest.raises(ValueError) – testuj, že nastane výjimka
  • @pytest.fixture – sdílený testovací vstup (DataFrame)
  • @pytest.mark.parametrize – jeden test, více vstupů
  • Spuštění: pytest nebo pytest -v (verbose)

Proč testovat?

Test = automatická kontrola „funguje moje funkce správně?". Bez testů zjistíš bug až když selže produkce. S testy ho odchytíš hned při změně kódu – regresní testování.

Základní test

# test_transformace.py
import pandas as pd
import pytest
from transformace import add_vat

def test_add_vat_basic():
    df = pd.DataFrame({'cena': [100, 200, 300]})
    result = add_vat(df, 'cena')
    assert result['cena_s_dph'].iloc[0] == 121.0

def test_add_vat_raises_on_missing_column():
    df = pd.DataFrame({'jiny': [1, 2]})
    with pytest.raises(KeyError):
        add_vat(df, 'cena')

Fixture – sdílená testovací data

@pytest.fixture
def sample_df():
    return pd.DataFrame({
        'cena': [100, 200, None],
        'kat': ['A', 'B', 'A']
    })

def test_no_nulls(sample_df):
    result = clean_nulls(sample_df)

Parametrize – více vstupů najednou

@pytest.mark.parametrize("cena, expected", [
    (100, 121.0),
    (200, 242.0),
    (0, 0.0),
])
def test_vat(cena, expected):
    df = pd.DataFrame({'cena': [cena]})
    assert add_vat(df, 'cena')['cena_s_dph'].iloc[0] == expected

Spuštění

pytest              # všechny testy
pytest -v           # verbose – jméno každého testu
pytest -k "test_vat" # jen testy s "test_vat" v názvu
⚠ Chytáky a záludné otázky
  • Proč název souboru musí začínat test_? – pytest automaticky hledá soubory podle vzoru test_*.py nebo *_test.py. Jinak soubor ignoruje.
  • Rozdíl unit test vs integrační test? – Unit test testuje jednu izolovanou funkci s umělými daty. Integrační test testuje spolupráci více komponent (načtení z DB + transformace + export).
  • Proč nepoužít jen print() pro kontrolu? – print() je manuální – musíš sám přečíst výstup. assert je automatický – pytest vypíše PASSED/FAILED a ukáže přesně, co neodpovídalo.
💡 Tip na 15 minut — říkej komisi Základ: klíčové slovo assert a konvence pojmenování. Test musí být funkce začínající test_ v souboru začínajícím test_. Testovací DataFrame dělám co nejjednodušší – s hodnotou 100, aby DPH (121) šlo ověřit z hlavy bez kalkulačky.
