Profilace výkonu Python skriptu

⏱ ~15 minut
📝 Potítko – napiš si na papír
  • Profilace = zjisti, která část kódu trvá nejdéle (bottleneck)
  • Rychlé: import time; start=time.time(); ...; print(time.time()-start)
  • Profesionální: python -m cProfile -s time skript.py (terminál!)
  • tottime = čas přímo ve funkci (bez podfunkcí) – hledej bottleneck zde
  • ncalls = kolikrát byla funkce zavolána

Proč profilovat?

Optimalizuj až po měření. Bez profilace nevíš, co skript zpomaluje. 80 % času bývá stráveno v 20 % kódu (Paretovo pravidlo). cProfile ti přesně ukáže kde.

A) Rychlé měření pomocí time

import time

start = time.time()
df = nacti_data()
df = transformuj(df)
end = time.time()
print(f"Trvalo: {end - start:.2f} sekund")

B) Profesionální profilace – cProfile (terminál)

# Spustit v TERMINÁLU – nemusíš měnit kód:
python -m cProfile -s time analyza.py

# Výstup (zkráceno):
#    ncalls  tottime  percall  cumtime  percall filename:lineno(function)
#      1000    2.345    0.002    3.120    0.003 transformace.py:45(clean)
#         1    0.001    0.001    3.121    3.121 analyza.py:1(<module>)

Jak číst výstup

SloupecCo znamená
ncallsKolikrát byla funkce zavolána
tottimeČas přímo v této funkci (bez podfunkcí) – hledej bottleneck zde
cumtimeCelkový čas včetně všech podfunkcí
percalltottime / ncalls

Přepínač -s (řazení výstupu)

python -m cProfile -s time analyza.py     # seřaď podle celkového času
python -m cProfile -s cumtime analyza.py  # kumulativní čas
python -m cProfile -s calls analyza.py   # počet volání
⚠ Chytáky a záludné otázky
  • V čem je cProfile lepší než time.time()? – cProfile nepotřebuje změny v kódu a profiluje celý skript najednou – zobrazí VŠECHNY funkce seřazené podle spotřeby. time.time() musíš ručně přidat kolem každého úseku.
  • Co je bottleneck? – Místo v kódu, které trvá neúměrně dlouho. Ve výstupu cProfile ho poznáš jako funkci s nejvyšším tottime.
  • Kde se spouští cProfile? – V TERMINÁLU: python -m cProfile. Ne v Pythonu jako funkce.
💡 Tip na 15 minut — říkej komisi Profilaci spouštím z terminálu: python -m cProfile -s time analyza.py. Přepínač -s time seřadí výsledky podle spotřebovaného času – bottleneck vidím hned nahoře. Klíčový sloupec je tottime: čas strávený přímo v té funkci bez podfunkcí. Na rozdíl od time.time() nemusím vůbec sahat do zdrojového kódu.
