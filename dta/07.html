Čtení dat z API a uložení do souboru

⏱ ~15 minut
📝 Potítko – napiš si na papír
  • API vrací data ve formátu JSON přes HTTP protokol
  • import requests; r = requests.get(url, timeout=10)
  • r.raise_for_status() – vyhodí výjimku při chybě 4xx/5xx (klíčový řádek!)
  • r.json() – parsuje JSON na Python dict/list
  • pd.json_normalize(data) – vnořený JSON do DataFrame
  • df.to_csv() nebo df.to_json(orient='records')

Co je REST API?

API (Application Programming Interface) umožňuje programu komunikovat s jiným programem. REST API vrací data ve formátu JSON přes HTTP. Příklad: GET https://api.kurzy.cz/v1/mena/ vrátí aktuální kurzy měn.

Čtení dat z API – kompletní příklad

import requests
import pandas as pd

url = "https://api.example.com/products"

response = requests.get(url, timeout=10)
response.raise_for_status()   # KLÍČOVÝ ŘÁDEK: hodí HTTPError při 4xx/5xx

data = response.json()
df = pd.DataFrame(data)

# Pro vnořené JSON:
df = pd.json_normalize(data['items'])

# Uložení:
df.to_csv('produkty.csv', index=False, encoding='utf-8-sig')
df.to_json('produkty.json', orient='records', force_ascii=False)

Autentizace – API klíč

headers = {"Authorization": "Bearer MUJ_KLIC"}
response = requests.get(url, headers=headers, timeout=10)

params = {"rok": 2024, "limit": 100}
response = requests.get(url, params=params)

Ošetření chyb

try:
    response = requests.get(url, timeout=10)
    response.raise_for_status()
    df = pd.DataFrame(response.json())
except requests.exceptions.Timeout:
    print("API neodpovědělo včas")
except requests.exceptions.HTTPError as e:
    print(f"Chyba HTTP: {e}")
⚠ Chytáky a záludné otázky
  • Proč raise_for_status() a ne if status_code == 200? – API vrátí chybu 404 jako validní HTTP odpověď. Bez raise_for_status() Python nespadne, snaží se parsovat chybovou stránku jako data a spadne až o kus dál na nesmyslné chybě.
  • Co je JSON? – JavaScript Object Notation – textový formát pro výměnu dat. V Pythonu odpovídá dict nebo list. response.json() ho automaticky deserializuje.
  • Rozdíl orient=records vs výchozí při to_json()? – records uloží jako list of dicts (každý řádek je objekt) – nejčastěji čitelný formát. Výchozí orientace je sloupcová.
💡 Tip na 15 minut — říkej komisi Nejdůležitější řádek: response.raise_for_status() hned po volání API. Bez něj mi kód nespadne při chybě 404 – místo toho se pokouší parsovat chybovou stránku jako datový zdroj a spadne o kus dál na nesmyslné chybě, která nemá s API nic společného. Tímto jedním řádkem mám vždy okamžité hlášení přesně tam, kde problém nastal.
