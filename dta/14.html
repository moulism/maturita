Transformace dat – datová kvalita a regex

⏱ ~15 minut
📝 Potítko – napiš si na papír
  • Datová kvalita: Správnost, Konzistence, Úplnost, Unikátnost
  • NaN = chybějící číslo (NumPy), None = záměrně prázdné, NULL = SQL
  • Ošetření: dropna(), fillna(mean), ffill() (časové řady)
  • Regex: \d číslo, \w znak, \s mezera, . cokoliv
  • Kvantifikátory: * (0+), + (1+), ? (0/1), {n,m}

Dimenze datové kvality

DimenzeCo znamenáPříklad problému
SprávnostData odpovídají byznysové definici sloupceV sloupci věk je hodnota −5 nebo 200
KonzistenceStejná data vypráví stejný příběh ve všech systémechAdresa zákazníka se liší v CRM vs účetnictví

Ošetření chybějících hodnot

df.isnull().sum()                          # počet NULL v každém sloupci
df.dropna(subset=['cena'])               # smaž řádky kde chybí cena
df['cena'].fillna(df['cena'].mean())     # nahrad průměrem
df['kategorie'].fillna('Neznámá')        # nahrad konstantou
df['teplota'].fillna(method='ffill')    # forward fill pro časové řady

⚠ Chytáky a záludné otázky
  • Rozdíl NaN, None, NULL? – None je Python objekt. NaN (Not a Number) je float hodnota NumPy pro chybějící numerická data. NULL je SQL. V Pandas je NaN standard pro chybějící data.
  • Kdy mazat NULL a kdy imputovat? – Maž (dropna) pokud chybí málo dat (<5 %) a ztráta nevadí. Imputuj (fillna průměrem) pokud nechceš ztratit řádky. ffill použij u časových řad.
  • Proč raw string r'' u regex? – Python by interpretoval \d jako escape sekvenci. S raw stringem r"\d" nechá backslash být a regex engine ho zpracuje správně.
💡 Tip na 15 minut — říkej komisi Datová kvalita má čtyři dimenze: správnost, konzistence, úplnost, unikátnost. Při práci s NULL vždy nejprve df.isnull().sum() – zjistím rozsah problému. Pak rozhodnu: málo chybí → smažu řádky, hodně chybí → imputuji průměrem nebo mediánem. Regex použiju pro validaci – PSČ musí mít přesně 5 číslic: r'^\d{5}$'.
