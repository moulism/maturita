Tvorba datových struktur pomocí jazyka DAX

⏱ ~15 minut
📝 Potítko – napiš si na papír
  • DAX = vzorcovací jazyk pro datový model (Power BI)
  • Measure = virtuální, nepočítá se dokud neklikneme, šetří RAM
  • Calculated Column = fyzicky uložený sloupec, vhodný pro filtrování
  • Calculated Table = nová tabulka z existujících dat
  • VAR/RETURN = proměnné pro čitelný a efektivní kód

Co je DAX a k čemu slouží

DAX (Data Analysis Expressions) je vzorcovací jazyk, který doluje analytiku z datového modelu. Připomíná pokročilý Excel, ale operuje nad celými tabulkami najednou.

Tři typy DAX výpočtů

Measure (Měření)
Virtuální — neuloží se, počítá se dynamicky při kliknutí na grafy. Šetří RAM. Příklad: CelkoveTrzby = SUM(Prodeje[Cena]). Vhodné pro 95 % výpočtů.
Calc. Column
Fyzicky uložený sloupec, přidaný k tabulce. Počítá se při aktualizaci dat. Příklad: Zisk = Prodeje[Cena] - Prodeje[Naklady]. Použít jen pokud potřebujeme filtrovat nebo tvořit hierarchie podle tohoto sloupce.
Calc. Table
Vytvoří novou tabulku z existujících dat. Příklad: Kalendář přes CALENDARAUTO(), nebo VIP zákazníci přes FILTER().

Measure vs. Calculated Column — kdy použít co

MeasureCalculated Column
UloženíNepočítá se, uloží se jen vzorecFyzicky uložen na disku a v RAM
VýkonŠetří RAM, doporučenoVelké tabulky → zpomalení modelu
Filtrování❌ Nelze filtrovat podle✅ Lze filtrovat
Kdy použítSoučty, průměry, poměry (95 % případů)Hierarchie, filtry, řazení

Proměnné — VAR a RETURN

Klíčová vlastnost DAX pro čitelný a výkonný kód. Bez proměnné se složitý výraz počítá tolikrát, kolikrát ho použijeme:

CistyZisk =
VAR Dan = 0.21
VAR Zaklad = SUM(Faktury[Cena])
RETURN
    Zaklad - (Zaklad * Dan)

VAR uloží výsledek do paměti jednou. RETURN ho pak použije vícekrát — mnohem rychlejší než opakovat výpočet.

Datové typy v DAX

Integer, Decimal, String, DateTime, Boolean (True/False). DAX neumí sčítat text a číslo — je nutná explicitní konverze pomocí funkcí jako VALUE() nebo FORMAT().

⚠️ Chytáky a záludné otázky
  • „Proč Measure místo Calculated Column?" — Measure se neuloží, nepočítá se, dokud neklikneme na graf → šetří RAM. Calculated Column se fyzicky uloží a počítá pro každý řádek → na velké tabulce zpomalí celý model.
  • „K čemu VAR/RETURN?" — Bez proměnné databáze počítá stejný výraz vícekrát. S VAR se složitý výpočet provede jednou a výsledek se uloží do paměti.
  • „Jaký je rozdíl CALCULATE a FILTER?" — CALCULATE mění kontext filtru (celý model). FILTER vrátí podmnožinu tabulky. CALCULATE se FILTER() uvnitř je velmi výkonná kombinace.
💡 Tip na 15 minut — říkej komisi„U DAX kódu zdůrazním důležitost proměnných VAR/RETURN. Bez nich se složitá suma počítá třikrát. S VAR se vypočítá jednou, uloží do paměti a pod RETURN se třikrát použije jako hotové číslo. Kód je čitelnější a výrazně rychlejší."
