Datové formáty a API (JSON, XML, CSV)

⏱ ~15 minut
📝 Potítko – napiš si na papír
  • CSV = text s oddělovačem, malý soubor, ale bez datových typů
  • JSON = klíč:hodnota, moderní API standard, lehký
  • XML = tagy, přesný a strukturovaný, ale "ukecaný"
  • API = most pro výměnu dat mezi aplikacemi (nejčastěji JSON)
  • ETL: Extract → Transform (80 % času) → Load
  • Power Query zpracovává CSV, JSON, API, Excel, SQL

API — Application Programming Interface

Bezpečný most, přes který si dvě aplikace vyměňují data, nejčastěji ve formátu JSON. Power BI se přes API napojí na ČNB a každý den si stáhne aktuální kurz eura.

ETL proces — Extract, Transform, Load

Extract
Připojení ke zdroji dat (API URL, soubor CSV, SQL databáze).
Transform
80 % práce! Určení datových typů, odstranění chyb, prázdných sloupců, nahrazení hodnot, rozbalení JSON struktur.
Load
Načtení čisté tabulky do BI modelu nebo datového skladu.
⚠️ Chytáky a záludné otázky
  • „Proč CSV nemá datové typy?" — CSV je jen text. Power Query musí hádat typy ze vzorku prvních 200 řádků. Anglický formát data v českém CSV nebo tečka místo čárky u čísla způsobí chybu importu.
  • „Proč XML má XSD schéma?" — XSD je „předpis" pro strukturu XML. Systém si dokáže data před uložením zkontrolovat — jsou povinná pole vyplněna? Jsou datové typy správné?
  • „Co je REST API?" — Architektura API postavená na HTTP protokolu. GET (čti), POST (vytvoř), PUT (uprav), DELETE (smaž). Odpověď nejčastěji v JSON formátu.
💡 Tip na 15 minut — říkej komisi„CSV je nejlehčí a nejrychlejší formát, ale skrývá riziko. Nemá datové typy — vše je jen text. Power Query hádá typy ze vzorku 200 řádků. Pokud se v českém CSV náhodou objeví anglické datum nebo cena s tečkou místo čárky, celý import se zastaví na fatální chybě."
