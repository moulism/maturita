SQL – DDL (Data Definition Language)

⏱ ~15 minut
📝 Potítko – napiš si na papír
  • DDL = definice struktury, nepracuje s daty uvnitř
  • CREATE TABLE = vytvoření | ALTER TABLE = změna | DROP = zničení
  • DROP smaže vše | TRUNCATE smaže data, zachová strukturu (nelze rollback!)
  • Datové typy: INT, DECIMAL(10,2), VARCHAR/NVARCHAR, DATE, DATETIME
  • Constraints: PRIMARY KEY, FOREIGN KEY, NOT NULL, UNIQUE, CHECK, DEFAULT

DDL příkazy a jejich účel

CREATE
Vytvoří databázi, schéma nebo tabulku. CREATE TABLE Studenti (ID INT, Jmeno NVARCHAR(50))
ALTER TABLE
Změní strukturu tabulky. ALTER TABLE Studenti ADD Telefon NVARCHAR(20) nebo DROP COLUMN Telefon
DROP
Zničí objekt kompletně — tabulka + data zmizí z disku. Nelze vrátit!
TRUNCATE
Agresivní vyprázdnění dat. Bleskově smaže všechna data a resetuje auto-increment. Struktura tabulky zůstane. Nelze vrátit (ROLLBACK)!
DROP vs. TRUNCATE vs. DELETE — kritický rozdíl!
  • DROP TABLE = zbourá celou tabulku i se strukturou (nevratné)
  • TRUNCATE TABLE = smaže všechna data, zachová strukturu, resetuje IDčka, nelze rollback, velmi rychlé
  • DELETE FROM (DML!) = maže řádek po řádku, loguje každé smazání, pomalé na velkých tabulkách, lze rollback

Constraints — omezení sloupců

CREATE TABLE Zakaznici (
    ID         INT PRIMARY KEY,          -- jedinečné, NOT NULL
    Email      NVARCHAR(100) UNIQUE NOT NULL, -- unikátní
    Vek        INT CHECK (Vek >= 18),   -- podmínka
    Stav       NVARCHAR(20) DEFAULT 'Aktivní', -- výchozí hodnota
    Mesto_ID   INT FOREIGN KEY REFERENCES Mesta(ID) -- cizí klíč
)
⚠️ Chytáky a záludné otázky
  • „Jaký je rozdíl VARCHAR a NVARCHAR?" — VARCHAR ukládá jen ASCII znaky (bez diakritiky). NVARCHAR ukládá Unicode (čeština, čínština) a zabírá 2× více místa. V ČR vždy NVARCHAR pro text s diakritikou!
  • „TRUNCATE vs. DELETE — kdy co použít?" — TRUNCATE: chci rychle vymazat vše (staging tabulka při ETL). DELETE: chci mazat konkrétní řádky s WHERE, nebo potřebuji zachovat možnost ROLLBACK.
  • „Co je FOREIGN KEY constraint?" — Zabrání vložení objednávky pro zákazníka, který neexistuje. Databáze kontroluje referenční integritu — bez FK by bylo možné mít objednávky pro neexistující zákazníky.
💡 Tip na 15 minut — říkej komisi„Studenti zaměňují DELETE, DROP a TRUNCATE. DROP smaže strukturu i data — z disku zmizí vše. DELETE maže řádek po řádku a pečlivě loguje — jde rollbackovat, ale je pomalé. TRUNCATE agresivně uřízne datovou část za milisekundu, resetuje IDčka, ale nejde vrátit. To je noční můra juniorních DBA."
