SQL – DML (Data Manipulation Language)

⏱ ~15 minut
📝 Potítko – napiš si na papír
  • SELECT pořadí: FROM → WHERE → GROUP BY → HAVING → SELECT → ORDER BY
  • INSERT INTO … VALUES | UPDATE … SET … WHERE | DELETE FROM … WHERE
  • UPDATE/DELETE bez WHERE = všechny řádky! Vždy s WHERE!
  • WHERE: AND, OR, BETWEEN, IN, LIKE (% = cokoliv, _ = jeden znak)
  • DISTINCT = unikátní | TOP/LIMIT = omezení | agregace: SUM, COUNT, AVG, MIN, MAX

DML příkazy

SELECT
Čtení dat. Nejpoužívanější příkaz v SQL.
INSERT INTO
Vkládá nové řádky. INSERT INTO Zakaznici (jmeno, email) VALUES ('Jan', 'jan@example.com')
UPDATE
Upravuje existující data. VŽDY s WHERE! UPDATE Zakaznici SET stav = 'VIP' WHERE id = 5
DELETE
Maže řádky. VŽDY s WHERE! DELETE FROM Zakaznici WHERE id = 5
Bez WHERE = katastrofa! UPDATE Produkty SET cena = 0 — vynuluje ceny VŠECH produktů!
DELETE FROM Zakaznici — smaže VŠECHNY zákazníky!

Pořadí zpracování dotazu SELECT

Databáze nezačíná od SELECT! Skutečné pořadí zpracování:

FROM → WHERE → GROUP BY → HAVING → SELECT → ORDER BY
Proto: ve WHERE nelze použít alias definovaný v SELECT (databáze ještě SELECT nezpracovala)! 
SELECT   mesto, COUNT(*) AS pocet   -- 5. co zobrazit
FROM     Zakaznici                    -- 1. odkud
WHERE    vek > 18                      -- 2. filtruj řádky
GROUP BY mesto                        -- 3. seskup
HAVING   COUNT(*) > 10              -- 4. filtruj skupiny
ORDER BY pocet DESC                  -- 6. seřaď

Operátory pro WHERE

AND / OR
AND: obě podmínky musí platit. OR: stačí jedna. Lze kombinovat se závorkami.
BETWEEN
WHERE cena BETWEEN 100 AND 500 — včetně hraničních hodnot.
IN
WHERE mesto IN ('Praha', 'Brno', 'Plzeň') — nahrazuje více OR podmínek.
LIKE
% = libovolný počet znaků. _ = přesně jeden znak. WHERE jmeno LIKE 'J%' = všechna jména začínající na J.
⚠️ Chytáky a záludné otázky
  • „Proč nesmí být agregace ve WHERE?" — WHERE běží PŘED GROUP BY. V tu chvíli žádné součty neexistují. Agregace patří do HAVING, které běží PO seskupení.
  • „Co vrátí SELECT * (hvězdička)?" — Všechny sloupce. V produkci se nedoporučuje — přenáší se zbytečná data přes síť. Vždy vyjmenovat konkrétní sloupce.
  • „Co se stane UPDATE bez WHERE?" — Aktualizuje VŠECHNY řádky v tabulce. Jedna z nejčastějších juniorních chyb s katastrofálními následky.
💡 Tip na 15 minut — říkej komisi„Studenti si myslí, že databáze čte dotaz od SELECT. Ale pořadí zpracování začíná od FROM — databáze potřebuje vědět, kam jít. Pak WHERE odfiltruje nepotřebná data, GROUP BY seskupí, HAVING odfiltruje skupiny a teprve pak SELECT rozhodne, co zobrazit. Proto alias definovaný v SELECT nelze použít ve WHERE — SELECT ještě neproběhl!"
