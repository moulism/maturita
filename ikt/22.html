Zpracování a vizualizace dat v IoT

⏱ ~15 minut
📝 Potítko – napiš si na papír
  • ESP8266 může fungovat jako web server → HTML stránka s teplotou
  • Node-RED = vizuální programování (flows/uzly) bez psaní kódu
  • Grafana = dashboard pro IoT data (vizualizace)
  • InfluxDB = time-series databáze (teplota + časové razítko) — ne MySQL!
  • TIG stack: Telegraf + InfluxDB + Grafana = profesionální IoT systém

Webová prezentace dat ze senzorů

ESP8266 umí fungovat jako malý HTTP server. Vytvoří webovou stránku zobrazující aktuální teplotu — připojíme se přes prohlížeč mobilu v místní síti. Pro rozsáhlejší systémy se data posílají na skutečný server (PHP + MySQL nebo InfluxDB).

Node-RED — vizuální programování IoT

Node-RED je vizuální programovací nástroj od IBM. Běží v prohlížeči (nejčastěji na Raspberry Pi). Téměř se nepíše kód — programuje se graficky pomocí „flows" (toků).

Příklad flow: [MQTT vstup] → přijme teplotu ze senzoru
[Logika IF] → pokud teplota > 25°C
[E-mail uzel] → pošli upozornění
[Grafana uzel] → zobraz v dashboardu

Použití: centrální mozek chytré domácnosti — propojuje žárovky Philips Hue, senzory, předpověď počasí, alarmy.

Grafana — vizualizace a dashboardy

Open-source SW pro vizualizaci dat. Sám data nesbírá — napojí se na databázi (InfluxDB, MySQL, Prometheus) a zobrazuje interaktivní dashboardy. Grafy, budíky, teploměry, alarmy. Ideální pro dohledová centra nebo správce serverů.

TIG Stack — profesionální IoT systém

Telegraf
Agent sbírající data ze senzorů a aplikací, posílá do InfluxDB.
InfluxDB
Time-series databáze — bleskový zápis měřených hodnot.
Grafana
Vizualizační vrstva — krásné interaktivní dashboardy.

Cloud a IoT platformy

ThingSpeak, Blynk, AWS IoT, Azure IoT Hub — cloudové platformy, kde senzor jen pošle data a platforma se postará o uložení, zabezpečení a vizualizaci. Nemusíme spravovat vlastní servery.

⚠️ Chytáky a záludné otázky
  • „Proč MySQL nevhodné pro IoT?" — IoT může generovat tisíce zápisů za sekundu (teplota, vlhkost, tlak). MySQL při každém zápisudělá složité indexování — pod takovým náporem se systém zhroutí. InfluxDB je navržena pro masivní zápis časových dat.
  • „Co je Node-RED a čím se liší od programování v Pythonu?" — Node-RED je vizuální — nepíše se kód, ale kreslí se grafický tok zpracování dat. Pro rychlé prototypy a lidi bez programátorského zázemí. Python je textové programování pro složitější logiku.
  • „Jaký je rozdíl Grafana a Power BI?" — Power BI: zaměřený na business analytiku, datové sklady, Excel data. Grafana: zaměřená na real-time monitoring (servery, IoT, metriky), time-series data, open-source.
💡 Tip na 15 minut — říkej komisi„Pro IoT monitoring se kombinuje TIG stack: data ze senzorů přitečou přes MQTT do Node-RED (validace dat), Node-RED je uloží do InfluxDB (bleskový zápis). Grafana se každou vteřinu dívá do InfluxDB a vykresluje na obrazovce správce krásné grafy. Profesionální, stabilní a snadno spravovatelný systém."
