Přenosové sítě, protokoly a sběrnice pro IoT

⏱ ~15 minut
📝 Potítko – napiš si na papír
  • Wi-Fi = velká spotřeba | BLE = malá spotřeba (hodiny) | Zigbee = MESH domácnost
  • LoRaWAN/Sigfox = 15 km, 10 let baterie, jen pár bajtů — SmartCity
  • HTTP = Request-Response, těžkopádný pro IoT
  • MQTT = Publisher/Subscriber přes Broker, lehký a efektivní
  • I2C = 2 dráty, 127 zařízení | SPI = rychlejší, více drátů | UART = 2 dráty, sériová linka

Jak MQTT funguje

Publisher → Broker → Subscriber
Senzor: „Publikuji: téma 'Dum/Teplota' = 22°C" → Broker
Aplikace se přihlásí k odběru tématu 'Dum/Teplota'
Při každé nové hodnotě Broker okamžitě rozešle zprávu všem odběratelům
Mobilní telefon, Grafana, domácí server — vše dostane update zároveň

Sběrnice — komunikace na desce

UART
Sériová linka. 2 piny (RX přijímač, TX vysílač). Nutná dohodnutá rychlost (Baud rate). GPS moduly, programování desek.
I2C
2 dráty (SDA data, SCL hodiny). Master/Slave s adresami. Na 2 dráty lze připojit 127 senzorů! Displeje, senzory.
SPI
Rychlejší než I2C. Více drátů (MOSI, MISO, SCK, CS). SD karty, displeje s rychlou grafiku.
OneWire
Jeden drát pro data i napájení (parazitní napájení). Typicky Dallas teplotní senzory (DS18B20).
⚠️ Chytáky a záludné otázky
  • „Proč MQTT místo HTTP v IoT?" — HTTP: mobil musí každou vteřinu dotazovat server (polling) — zbytečná zátěž sítě a baterie. MQTT: mobil se přihlásí k odběru a server pošle data sám, jakmile se změní. Efektivnější, šetří baterii i síť.
  • „Co je MESH síť Zigbee?" — Každé Zigbee zařízení může fungovat jako router a přeposílat zprávy dalším zařízením. Pokud jedna žárovka ztratí signál, zpráva se „obejde" přes jinou žárovku. Velmi robustní pro domácí automatizaci.
  • „Kdy LoRaWAN vs. Wi-Fi?" — LoRaWAN: senzor na poli 10 km od civilizace, měřící půdní vlhkost s 10letou baterií → posílá jen číslo jednou za hodinu. Wi-Fi: IP kamera v domě přenášející video → potřebuje vysoký datový tok a napájení ze zásuvky.
💡 Tip na 15 minut — říkej komisi„MQTT vs. HTTP: kdybychom měli teplotní senzor s HTTP, mobil by se musel serveru neustále ptát 'Změnila se teplota? Změnila se?' každou vteřinu. U MQTT se mobil jednou přihlásí k odběru tématu. Až senzor naměří novou hodnotu, broker ji sám rozešle všem odběratelům. Šetří baterii i šířku pásma sítě."
