Programování IoT zařízení (Arduino/ESP)

⏱ ~15 minut
📝 Potítko – napiš si na papír
  • Jazyk: C++ (zjednodušený), IDE = Arduino IDE
  • void setup() = jednou při startu | void loop() = nekonečná smyčka
  • pinMode() = nastavení pinu | digitalWrite() = výstup | digitalRead() = vstup
  • delay(ms) = pauza programu | millis() = čas od startu (non-blocking!)
  • Serial.print() = ladění přes USB | #include = import knihovny

Programovací jazyk a struktura

Pro Arduino a ESP desky se používá jazyk vycházející z C a C++, zjednodušený o předpřipravené funkce pro práci s hardwarem. Každý program (Sketch) musí obsahovat dvě základní funkce:

FunkceKdy se spustíTypický obsah
void setup()Jednou při startu nebo resetuNastavení rychlostí portů, režimů pinů
void loop()Nekonečná smyčka (tisíckrát za sekundu)Čtení senzorů, řízení výstupů, odesílání dat

Základní příkazy pro I/O piny

pinMode(pin, mode)
Nastavuje pin jako vstup (INPUT) nebo výstup (OUTPUT). Píše se do setup().
digitalWrite(pin, val)
Nastaví výstupní pin na HIGH (5V/zapnuto) nebo LOW (0V/vypnuto). Pro LED, relé.
digitalRead(pin)
Přečte stav vstupního pinu. Vrátí HIGH nebo LOW. Pro tlačítka.
analogRead(pin)
Přečte analogovou hodnotu (0–1023). Pro fotorezistory, potenciometry.
analogWrite(pin, val)
PWM výstup (0–255). Stmívání LED, řízení rychlosti motoru.

Ukázkový program — blikající LED se senzorem

#include <DHT.h>                  // import knihovny pro senzor
DHT dht(2, DHT11);               // senzor na pin 2
const int ledPin = 13;

void setup() {
    dht.begin();
    pinMode(ledPin, OUTPUT);
    Serial.begin(9600);          // pro Serial Monitor
}

void loop() {
    float temp = dht.readTemperature();
    Serial.println(temp);         // zobraz v terminálu
    
    if (temp > 25.0) {
        digitalWrite(ledPin, HIGH);  // rozsvít LED
    } else {
        digitalWrite(ledPin, LOW);
    }
    delay(1000);                    // čekej 1 sekundu
}

delay() vs. millis() — non-blocking

delay() ZASTAVÍ celý program! Pokud Arduino čeká delay(5000) = 5 sekund, nemůže během té doby nic jiného dělat. Tlačítko stisknuto uprostřed delay() = zaignorováno!

millis() vrátí počet ms od startu desky. Pomocí porovnání (if (millis() - lastTime > 5000)) lze dělat více věcí zároveň bez blokování.

Arduino IDE — nastavení

Vývojová deska
Nástroje → Vývojová deska → vybrat Arduino Uno / ESP32 / ESP8266. Nutné doinstalovat podporu pro ESP!
Port
Nástroje → Port → vybrat správný COM port. Bez správného portu kód nelze nahrát.
Serial Monitor
Okno pro komunikaci s deskou přes USB. Serial.print() zobrazuje debugovací zprávy a naměřené hodnoty.
⚠️ Chytáky a záludné otázky
  • „Proč delay() je problém v reálných projektech?" — delay() blokuje celý procesor. Pokud čekám 5 sekund, tlačítko stisknuto v tu dobu = nezaregistrováno. Řešení: millis() — non-blocking čekání (stejný princip jako setTimeout v Javascriptu).
  • „Jaký je rozdíl digitalRead a analogRead?" — digitalRead: pouze HIGH (5V) nebo LOW (0V) — binární. analogRead: hodnota 0–1023 odpovídající napětí 0–5V. Fotorezistor mění odpor podle světla → napětí se mění → analogRead vrátí číslo od 0 do 1023.
  • „Co jsou knihovny a proč je používáme?" — Hotový kód pro komplikovaný hardware (DHT11 senzor, servo, OLED displej). Bez knihovny bychom museli naprogramovat složitý komunikační protokol pro každý senzor od nuly. #include = import knihovny.
💡 Tip na 15 minut — říkej komisi„V Arduino skriptu se pohyb postavy ve hře nesmí řídit příkazem 'posun o 5m' v každém volání loop(). Na pomalém PC s 30 FPS = 150 m/s, na rychlém s 200 FPS = 1000 m/s. U Unity je řešení Time.deltaTime, u Arduina je analogie millis() — počítáme čas od posledního pohybu a pohybujeme o hodnotu úměrnou uplynulému času."
